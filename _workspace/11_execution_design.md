# 네이버 스포츠 마이팀 전략: 실행 설계 데이터

> 조사일: 2026-04-08
> 목적: 실행 가능성(3/5 -> 5/5), 실행 로드맵(3.5/5 -> 5/5) 달성을 위한 구체적 데이터

## 1. 네이버 조직 구조 (스포츠 관련)

### 1.1 2024년 4월 조직개편: CIC -> 12개 전문조직

네이버는 2024년 4월 기존 5개 CIC(사내독립기업)를 12개 전문조직으로 세분화했다. 이는 9년 만의 대규모 개편이다.

**기존 5개 CIC:**
| CIC | 담당 영역 |
|-----|-----------|
| 서치 | 검색 서비스 |
| 커뮤니티 | 창작자 지원, 카페, 밴드 |
| 비즈 | PC/모바일 검색, 디스플레이 광고 |
| 포레스트 | 네이버 쇼핑, 스마트스토어 |
| 글레이스 | 네이버 예약, 지도, 플레이스 |

**개편 후 3대 영역:**
1. **프로덕트 & 플랫폼 영역**: 사용자 경험/기술 혁신 중심 개발 설계
2. **비즈니스 & 서비스 영역**: 비즈니스 기회 발굴, 서비스 매력 제고
3. **콘텐츠 영역**: 사용자 니즈 맞춤 콘텐츠 유형 개발/제공

**독립 Cell 조직**: 치지직, 밴드, 뮤직 (기민한 독립 성장 목적)

**거버넌스**: 최수연 대표 직속 3개 위원회 (글로벌경영, 프로덕트-테크, 임직원성장)

### 1.2 스포츠 서비스 조직 위치 (추정)

공식적으로 스포츠 전담 조직명은 비공개이나, 조직 구조상 다음과 같이 매핑된다:

| 마이팀 관련 기능 | 관련 조직 (추정) | 협업 필요도 |
|-----------------|-----------------|------------|
| 스포츠 페이지/뉴스 | 콘텐츠 영역 (구 서치CIC 일부) | 핵심 |
| 마이팀 알림 시스템 | 프로덕트 & 플랫폼 영역 | 핵심 |
| AI 개인화 추천 | 프로덕트 & 플랫폼 영역 (AI 조직) | 핵심 |
| 커뮤니티 연동 (카페/오픈톡) | 커뮤니티 Cell 또는 전문조직 | 높음 |
| 치지직 스포츠 중계 | 치지직 Cell (독립) | 높음 |
| 네이버플러스 멤버십 연계 | 비즈니스 & 서비스 영역 | 중간 |
| 광고/수익화 | 비즈니스 & 서비스 영역 | 중간 |

### 1.3 실행 설계 시사점

- **Cross-functional 조율 필요**: 마이팀 고도화는 최소 4~5개 전문조직의 협업이 필요
- **프로덕트-테크 위원회 활용**: 대표 직속 위원회를 통한 우선순위 확보가 성공 열쇠
- **Cell 조직과의 협업 프로토콜**: 치지직은 독립 Cell로, 별도 MOU/OKR 합의 필요

- **트래픽 스파이크**: KBO 주요 경기 시 동시접속자 급증 대응 (10배 이상 트래픽 버스트)

- **개인화 필터링**: 마이팀 기반 사용자별 맞춤 알림 라우팅

**복잡도 평가:**

| 선수 소식 개인화 | 높음 | 낮음 | 선수 DB+NLP |

## 3. Go/No-Go 기준표

### 3.1 Phase-Gate 프레임워크

각 Phase의 Gate에서 Go/No-Go/Hold/Conditional Go 결정을 내린다.

```
Phase 0 (Discovery) --[Gate 0]--> Phase 1 (Foundation) --[Gate 1]--> Phase 2 (Enhancement) --[Gate 2]--> Phase 3 (AI/Premium)
```

### 3.2 KPI 벤치마크 기반 Go/No-Go 기준

#### A. 알림(Push Notification) CTR 벤치마크

| 지표 | 업계 평균 | 스포츠 앱 평균 | 개인화 적용 시 | 마이팀 목표 |
|------|----------|--------------|-------------|-----------|
| 전체 CTR | 2.25% | 0.83% (iOS) / 1.69% (Android) | ~8% (SPORT1 사례) | Phase1: 3%, Phase2: 5%, Phase3: 8% |

| 비활성화율 | - | - | - | < 5%/월 |

### 3.3 Phase별 Go/No-Go 의사결정 기준표

#### Gate 0: Discovery -> Foundation 전환

| 기준 | Go | Conditional Go | No-Go |
|------|-----|---------------|-------|
| 마이팀 설정 유저 수 | > 100만 | 50만~100만 | < 50만 |
| 기술 PoC 완료 | 전체 완료 | 80% 이상 | < 80% |
| 예산 확보 | 100% | 80% 이상 | < 80% |
| 핵심 인력 확보 | PM+TL+ML리드 확보 | PM+TL 확보 | PM 미확보 |

> 참고: 2026.04 기준 마이팀 누적 등록 ~336만 (10개 구단 합산) -- 이미 Gate 0 Go 조건 충족

### 4.1 리스크 평가 프레임워크

**발생 확률 (Likelihood):**
| 등급 | 점수 | 정의 |
|------|------|------|
| 매우 낮음 | 1 | < 10% 확률 |
| 낮음 | 2 | 10~30% |
| 중간 | 3 | 30~50% |
| 높음 | 4 | 50~70% |
| 매우 높음 | 5 | > 70% |

**영향도 (Impact):**
| 등급 | 점수 | 정의 |
|------|------|------|
| 무시 | 1 | 일정 1주 이내 지연, 비용 영향 < 5% |
| 경미 | 2 | 일정 2주 지연, 비용 5~10% |
| 보통 | 3 | 일정 1개월 지연, 비용 10~20% |
| 심각 | 4 | 일정 2개월+ 지연, 비용 20~40% |

## 8. 알림 피로 완화 전략 (상세)

마이팀 전략에서 알림은 핵심 인게이지먼트 도구이자 최대 리스크 요인이다.

### 8.1 벤치마크 기반 설계 원칙

| 원칙 | 근거 | 적용 |
|------|------|------|
| 일 최대 3회 제한 | 업계 권장 빈도 캡 | 경기일에도 3회 이내 |
| 개인화 필수 | 개인화 시 CTR 3~4배 향상 | 마이팀/선수 기반 필터링 |
| 다이제스트 옵션 | 개별 알림 대신 요약 알림 | "오늘의 마이팀 요약" 알림 |
| Quiet Hours 존중 | 밤 10시~아침 7시 알림 보류 | 사용자 설정 가능 |
| 세분화된 설정 | 사용자 통제권 부여 | 경기/뉴스/커뮤니티별 on/off |

### 8.2 모니터링 KPI

| 지표 | 경고 기준 | 위험 기준 | 대응 |
|------|----------|----------|------|
| 알림 opt-out율 | 월 3%+ | 월 5%+ | 빈도 자동 감소 |
| 알림 CTR 하락 | 전주 대비 -20% | 전주 대비 -40% | 콘텐츠/타이밍 재검토 |
| 앱 삭제율 (알림 기인) | 월 1%+ | 월 2%+ | 긴급 알림 정책 변경 |
