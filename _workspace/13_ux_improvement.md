# 네이버 스포츠 마이팀 UX 개선 보고서

> **목적**: 마이팀 전환율 7.5% → 15~20% 달성을 위한 UX 개선안
> **대상**: UX 디자이너 (Figma 시안 제작용 명세 포함)
> **기준 디바이스**: iPhone 14 (390 x 844pt)
> **브랜드 컬러**: 네이버 그린 #03C75A / 서브 #00B843 / 배경 #F7F8FA

## 1. 현재 마이팀 진입 경로 전수 조사

### 1.1 경로 A: 스포츠 탭 → 메뉴 → 마이팀 설정 (주요 경로)

| 단계 | 화면/행동 | 예상 이탈률 |
|------|----------|------------|
| 1 | 네이버 앱 실행 | 0% |
| 2 | 하단 탭 바 or 더보기(···) → "스포츠" 서비스 진입 | 15% (스포츠 탭이 기본 노출되지 않는 유저) |
| 3 | 스포츠 페이지 우측 상단 햄버거 메뉴(≡) 탭 | 25% (메뉴 위치 인지 실패) |
| 4 | "응원하는 팀을 구독해 보세요" 버튼 탭 | 20% (CTA 인지 실패) |
| 5 | 종목 선택 → 구단 "+" 버튼 탭 → "저장" | 15% (다단계 선택 피로) |
| 6 | 마이팀 페이지 진입 → 알림 설정 활성화 | 30% (별도 알림 설정의 존재 자체를 모름) |

**총 클릭 수**: 최소 6~8회
**누적 이탈률 추정**: ~70% (1000명 진입 시 ~300명만 완료)

### 1.2 경로 B: 스포츠 탭 → KBO/리그 페이지 → 팀 페이지 → 마이팀 추가

| 단계 | 화면/행동 | 예상 이탈률 |
|------|----------|------------|
| 1 | 네이버 앱 → 스포츠 진입 | 15% |
| 2 | KBO 또는 특정 리그 탭 선택 | 10% |
| 3 | 순위표/경기 목록에서 특정 팀 탭 | 10% |
| 4 | 팀 상세 페이지에서 "마이팀 추가" 버튼 발견 및 탭 | 35% (버튼 위치 인지 어려움) |
| 5 | 알림 설정 별도 진행 | 30% |

**총 클릭 수**: 최소 5~7회
**누적 이탈률 추정**: ~65%

### 1.3 경로 C: 경기 중계 페이지 → 마이팀 추가

| 단계 | 화면/행동 | 예상 이탈률 |
|------|----------|------------|
| 1 | 네이버 앱 → 스포츠 → 라이브 경기 진입 | 10% |
| 2 | 중계 페이지에서 팀 로고 또는 팀명 탭 | 20% |
| 3 | 팀 상세 페이지 이동 → 마이팀 추가 | 35% |
| 4 | 알림 설정 별도 진행 | 30% |

**총 클릭 수**: 최소 4~6회
**누적 이탈률 추정**: ~63%

### 1.4 경로 D: 네이버 검색 → 스포츠 영역 → 마이팀 추가

| 단계 | 화면/행동 | 예상 이탈률 |
|------|----------|------------|
| 1 | 네이버 검색 "KIA 타이거즈" | 0% |
| 2 | 검색 결과 스포츠 영역에서 팀/경기 정보 확인 | 10% |
| 3 | 스포츠 상세 페이지로 이동 | 25% |
| 4 | 마이팀 추가 버튼 발견 및 탭 | 35% |
| 5 | 알림 설정 별도 진행 | 30% |

**총 클릭 수**: 최소 4~6회
**누적 이탈률 추정**: ~66%

### 1.5 알림 설정의 치명적 복잡성

마이팀을 추가한 후에도 실제 알림을 받으려면 **4개의 개별 토글**을 모두 켜야 한다:

1. 네이버 설정 > 알림 수신 설정 > **전체 푸시 알림 ON**
2. 네이버앱 설정 > 스포츠 알림 > **푸시 알림 ON**
3. 네이버앱 설정 > 스포츠 알림 > 구단 알림받기 목록 > **MY팀 알림 ON**
4. 네이버앱 설정 > 스포츠 알림 > **관심 구단 경기예고, 경기결과 ON**

이 복잡성이 SNS에서 "알림 설정법 가이드" 콘텐츠가 대량 유통되는 근본 원인이다. TikTok, Threads, Instagram에서 "네이버 야구 마이팀 설정하는 방법"이 인기 검색어로 지속 노출되고 있으며, 이는 UX가 직관적이지 않다는 강력한 시그널이다.

## 3. 모바일 앱 온보딩 UX 베스트 프랙티스

### 3.1 핵심 통계 데이터

| 지표 | 수치 | 출처 |

### 3.2 1탭/2탭 온보딩의 전환율 차이

**핵심 원칙**: 설치 → 핵심 행동(팀 팔로우) 사이의 단계를 최소화할수록 전환율이 기하급수적으로 상승한다.

- **1탭 전환**: 마찰 최소, 충동적 행동 포착에 최적. Apple Pay/Google Pay 사례에서 검증.
- **2~3탭 전환**: 최소한의 맥락(종목 선택 → 팀 선택)이 필요한 경우 허용 가능한 범위.
- **5탭 이상**: 완료율이 급격히 하락하며, 외부 가이드(SNS 설정법 공유)가 필요해지는 임계점.

### 3.3 Progressive Disclosure 패턴

핵심 행동만 먼저 노출하고, 세부 설정은 나중에 점진적으로 안내하는 방식.

**적용 방안**:
- 1차: 팀 선택만 요청 (1탭)
- 2차: 알림 수신 여부 선택 (팀 선택 직후 자동 프롬프트)
- 3차: 세부 알림 커스텀은 마이팀 페이지에서 나중에

### 3.4 맥락적 프롬프트 (Contextual Prompt) 패턴

사용자가 특정 맥락에 있을 때 관련 액션을 제안하는 방식.

**적용 시나리오**:
- 경기 중계 시청 중 → "이 팀을 마이팀으로 추가하시겠어요?"
- 경기 결과 확인 후 → "다음 경기 알림을 받으시겠어요?"
- 팀 관련 검색 시 → 검색 결과에 마이팀 추가 버튼 노출

### 3.5 Bottom Sheet / FAB 패턴

- **Bottom Sheet**: 현재 화면을 유지하면서 추가 액션을 제공. 모달 대비 25~30% 높은 참여율.
- **FAB (Floating Action Button)**: 스크롤 중에도 항상 접근 가능한 CTA. 엄지 영역(thumb zone)에 최적화.

## 4. 마이팀 UX 개선안 (터치포인트별)

---

### 4-A. 네이버 앱 스포츠 탭 첫 진입 시 (마이팀 미설정 유저)

#### AS-IS (현재 UX)
- 스포츠 탭 진입 시 모든 유저에게 동일한 일반 스포츠 뉴스/스코어 화면 노출
- 마이팀 설정 유도 요소 없음 (또는 햄버거 메뉴 내부에 숨겨져 있음)
- 유저가 능동적으로 메뉴를 탐색해야만 마이팀 기능 발견 가능

#### TO-BE (개선 UX)
**전략**: Full-Screen Onboarding Bottom Sheet + 1탭 팀 선택

스포츠 탭 **최초 1회** 진입 시, 화면 하단에서 올라오는 Bottom Sheet로 팀 선택 온보딩을 즉시 노출한다.

#### 컴포넌트 상세 명세

- 오버레이: 뒤 콘텐츠에 #000000 40% 딤 처리

**타이틀 영역**

## 7. 구현 우선순위 로드맵

### Phase 1: Quick Win (2~4주)
1. **4-F 알림 설정 간소화** — 기존 336만 마이팀 유저의 알림 활성화율 즉시 개선
